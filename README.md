# MOP — Mimirhead Operations Platform (v2)

Внутренняя операционная платформа студии аутсорса игрового арта:
слой аналитики, биллинга и HR поверх производственных данных Kaiten.

## Стек

Next.js 15 (App Router, TS strict) · Supabase (Postgres, Auth) · Vercel · Tailwind + shadcn/ui

## Принципы (выстраданы v1)

1. Пользовательский доступ — только через RLS; service role живёт исключительно в cron-синке.
2. Ручные данные никогда не хранятся в зеркале `kaiten.*`.
3. Деструктивные операции синка — только внутри транзакционных Postgres-функций.
4. Миграции append-only, применяются CI: staging на PR, prod на merge в `main`.
5. Ошибки синка видимы в UI; «0 записей» — предупреждение, а не успех.
6. Ни одной таблицы «на будущее»: сущность создаётся вместе с фичей, которая её читает.
